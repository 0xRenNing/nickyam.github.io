macOS beta更新提示空间不足，无限循环重启问题解决

2019.10.04 虞侃 ⓶科技  热度 ℃

今天早上出门前，我下载了macOS Catalina beta新系统，让它自己更新。下午回来的时候发现系统更新失败，提示储存空间不足。以往碰到这种情况，我重启电脑后进入旧系统，释放部分空间后重新更新。

这一次的情况有所不同。一方面，更新前我的空余硬盘空间还有20GB，按照常理不应该提示空间不足；另一方面，更新失败后无法进入旧系统，无限循环更新系统，无限提示更新失败。

理论上，我的电脑已经彻底歇菜了。如何解决这个问题，进入愿系统呢？

第一步，我通过 option 选择 引导系统，试图进入愿系统。但是进度条推进到一半的时候，屏幕一闪直接黑屏退出自动关机。疑似硬件都有问题了。

第二步，我准备检查硬件是否有问题。因为我的电脑安装了Boot Camp，所以我通过 option选择进入 Windows 10 系统，一切正常。这个系统我设置了 35GB 的空间，剩余空间 4.65GB 。因此可以确定硬件基本正常。

第三步，也是比较关键的一步。我通过 command + r 进入恢复系统。因为我的主分区被认定为空间不足，所以我就在实用工具选择了磁盘工具，将 Boot Camp 磁盘抹掉并合并到主分区。这样，我的磁盘剩余空间有50GB以上，足够更新了。

我唯一的担心是在恢复系统界面重新安装macOS之后，可能会完全抹掉我的个人数据。幸好，经过将近一个小时的更新与等待，我进入了熟悉的界面和新的操作系统。

就这样，我解决了标题所说的问题，但是牺牲了平时不常用的一个系统。虽然有些可惜，但是已经是不幸中的万幸了。
